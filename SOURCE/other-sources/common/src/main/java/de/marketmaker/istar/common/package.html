Packages/Klassen, die in anderen Modulen verwendet werden, i.d.R. wird eine eng begrenzte Funktionalität zur Verfügung gestellt, die von allgemeinem Nutzen ist.
compile
Kompilierung von Java-Code zur Laufzeit
http
Wrapper für commons-httpclient zur einfachen Verwendung http-basierter Schnittstellen
io
DataFile: eine Datei, die per FileChannel (nio) manipuliert wird
jmx
Hilfsklassen zur Integration eigener Klassen in JMX-Schnittstellen
lifecycle
Phasen im Lebenszyklus eines Objekts (Beans)
log
JmxLog um eine Zeile per JMX in das Applikationslog zu schreiben.
mcast
Multicast-Pakete senden und empfangen
mm
Klassen zur Anbindung eines portfolio managers (Java-Teil, native Methoden werden per C++ realisiert (istar/common/src/main/cpp)
mock
Test des MMService. ???
monitor
Überwachung von Resourcen (Files, Directories) und Auslösen von Ereignissen, wenn sich diese Resourcen ändern
nioframework
tcp-basierter Server, der non-blocking IO einsetzt (d.h. 1 Selector-Thread, der die angebundenen Clients verwaltet)
pool
Pool von DataChunk-Objekten, die letzlich byte[] enthalten und sich verketten lassen
request
Grundlegende Interfaces/Top-Level-Klassen für Remote Service Aufrufe/Antworten
rmi
obsolet
spring
Hilfsklassen für Spring-basierte Applikationen: Main ist unsere allgemeine Startklasse, die einen ApplicationContext anlegt und dazu von de.marketmaker.istar.launcher.spring.Launcher angelegt wird.
statistics
Erfassung von Statistiken
util
Sammlung eigenständiger Hilfsklassen

Validierung von Objekten basierend auf Annotationen. In der Regel verwendet, um Command-Objekte (spring CommandController) zu validieren, deren Properties auf Basis von HTTP-Request-Parametern gefüllt werden
